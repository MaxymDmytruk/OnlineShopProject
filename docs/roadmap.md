## Roadmap 

Цей документ описує приблизний план розробки "E-Shop Core API" на найближчі тижні.

### Тиждень 3: CRUD для Товарів (Products)

* **Мета:** Створити повний набір API-ендпоїнтів для управління товарами.
* **Завдання:**
    * Спроектувати модель `Product` (Товар) у базі даних (SQLite).
    * Реалізувати **POST /products** (Створення нового товару).
    * Реалізувати **GET /products** (Отримання списку всіх товарів).
    * Реалізувати **GET /products/{id}** (Отримання деталей одного товару).
    * Реалізувати **PUT /products/{id}** (Повне оновлення товару).
    * Реалізувати **DELETE /products/{id}** (Видалення товару).

### Тиждень 4: Базова Авторизація (Users & Auth)

* **Мета:** Дозволити користувачам реєструватися та входити в систему.
* **Завдання:**
    * Спроектувати модель `User` (Користувач) із хешованим паролем.
    * Реалізувати ендпоїнт **POST /register** (Реєстрація).
    * Реалізувати ендпоїнт **POST /login** (Вхід), який повертає токен (наприклад, JWT).
    * Захистити "адмінські" ендпоїнти (наприклад, створення товару) так, щоб вони вимагали цей токен.

### Тиждень 5: Замовлення (Orders)

* **Мета:** Дозволити авторизованим користувачам "купувати" товари.
* **Завдання:**
    * Спроектувати модель `Order` (Замовлення), яка пов'язує `User` та `Product`.
    * Реалізувати ендпоїнт **POST /orders** (Створення замовлення / "покупка").
    * Реалізувати ендпоїнт **GET /my-orders** (Дозволяє покупцю побачити список своїх придбаних товарів).
