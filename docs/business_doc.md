

### Назва системи: E-Shop Core API (ESC API)

ESC API — це бекенд-система для онлайн-магазину. Вона забезпечує базову функціональність для управління товарами та обробки замовлень. Система побудована на архітектурі API, що дозволяє легко підключати до неї різні клієнтські додатки (веб-сайт, мобільний додаток). На початковому етапі фокусується на каталозі товарів.

### Користувачі (ролі):
1.  **Покупець (Customer):** Неавторизований або авторизований користувач, який переглядає каталог та, у майбутньому, робить замовлення.
2.  **Адміністратор (Admin):** Авторизований користувач, який керує каталогом (додає/редагує/видаляє товари) та обробляє замовлення.

### Основні сценарії використання: 
1.  **Перегляд каталогу:** Покупець відкриває головну сторінку та бачить список доступних товарів.
2.  **Детальний перегляд:** Покупець переглядає повний опис та характеристики конкретного товару.
3.  **Керування товарами (Admin):** Адміністратор авторизується та додає новий товар до бази даних.
4.  **Пошук товару:** Покупець вводить назву товару у пошуковий рядок та отримує список релевантних результатів.

### Функціональні вимоги:
* Система повинна надавати API-ендпоінт для отримання списку всіх товарів (`GET /items`).
* Система повинна надавати API-ендпоінт для отримання одного товару за його ID (`GET /items/{id}`).
* Система повинна дозволяти Адміністратору додавати, редагувати та видаляти товари (CRUD).
* Система повинна надавати можливість пошуку товарів за назвою або описом.
* Система повинна забезпечувати з'єднання з базою даних.

### Нефункціональні вимоги:
* **Продуктивність:** Час відповіді на запит `/items` не повинен перевищувати 500 мс.
* **Безпека:** Admin-ендпоінти повинні бути доступні лише після успішної авторизації.
* **Надійність:** Система повинна використовувати логіювання для запису помилок.
* **Зручність:** API повинен повертати дані у стандартному форматі JSON.
* **Масштабованість:** Архітектура повинна бути побудована таким чином, щоб у майбутньому можна було легко додати модулі (наприклад, "Payments").

### Обмеження:
* Система не буде обробляти платежі або інтегруватися з платіжними шлюзами на початковому етапі.
* Система не буде мати складних механізмів доставки чи розрахунку податків.
